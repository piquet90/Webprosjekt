Teknologier brukt

HTML:

Vi skal bruke HTML fordi det er det nettleseren leser, og fordi det er et krav at vi må ha det med i oppgaven.

CSS:

Vi skal bruke CSS fordi det er den letteste måten å designe en web-side på og er også veldig mye muligheter med det. I tillegg er CSS et krav, og derfor må det være med.

JavaScript:

JavaScript blir brukt i oppgaven fordi det er en fin måte å gjøre siden mer interaktiv og moderne på. I tillegg er dette også et krav i oppgaven.

AJAX

AJAX gjør siden veldig dynamisk og helhetlig, fordi vi kan oppdatere deler av siden uten å oppdatere hele siden. Dette vil også vise en mestring av JavaScript, siden JavaScript er sentralt i AJAX. Dette er grunnene til at vi har valgt å ha AJAX med i oppgaven

PHP

Vi har valgt å bruke PHP fordi dette gir oss en del flere muligheter i tillegg til teknologiene over, samt at vi også får brukt disse teknologiene på flere måter enn hvis vi ikke hadde brukt PHP. PHP kan også brukes til å snakke med databaser, noe som gjør at vi kan bruke en database i prosjektet.

MySQL

Vi skal bruke MySQL som database fordi dette er en av de mest brukte databasene og fordi vi i teorien skal ha tilgang til dette på skolen. Det at vi har valgt å ha en database i oppgaven åpner for mange flere muligheter med tanke på lagring av data som brukerens penger, erfaringspoeng og profibilde. Vi kan da også opprette et pålogginssystem som er helt nødvendig for at ideen vår skal funke. Ellers er en database generelt greit med tanke på lagring av all slags data som skulle komme til å trenge.

GitHub

Vi har valgt å bruke GitHub i dette prosjektet. Dette har vi gjort fordi GitHub gjør det veldig lette for oss å sammarbeide på nettsiden. GitHub gir oss også en historikk av forandringer som er gjort i prosjektet. En annen grunn til at vi har valgt å bruke GitHub er at vi kommer til å ha mye bruk for dette i senere tid, og vi føler da at det er greit å begynne å bruke og lære det allerede nå.
